Platform Overview

3W+ is CelcomDigi's internal RAN analytics and site-planning platform. It consolidates congestion monitoring, CAPEX planning, AI-assisted querying, and GIS-based field-planning workflows into a single web application accessible from any browser.

RAN Forecast

Sector-level congestion, PRB utilisation, and 52-week ML forecasts across the full network.

Interactive Map

MapLibre GL map with coverage overlays, road-routing, CCTV siting, indoor simulation, and draw tools.

AI Chat Agent

Natural-language querying of network data via LangGraph + Claude, with RAG over uploaded documents.

Projects

Kanban board for network planning tasks with due dates, status tracking, and map-linked annotations.

CAPEX Planner

Upgrade-case breakdowns, region summaries, and per-site CAPEX estimates.

Data Management

Upload and pipeline raw CSV/Parquet datasets. Trigger ETL stages individually or as a full run.

3W+ runs as a containerised web service. Access it via your organisation's internal URL. No installation is required — a modern browser (Chrome, Edge, Firefox) is sufficient.

Getting Started

Logging In

1
Open the platform URL in your browser (e.g. https://3w.celcomdigi.internal/). You will be redirected to the login page if not authenticated.
2
Enter your username and password.

Credentials are provisioned by your Super Admin. Contact IT if you need an account.

3
Click Log In.

You are redirected to the RAN Forecast dashboard. The navigation bar at the top shows all modules available to your role.

User Roles

Three roles control feature access:

  • User — Read-only analytics, map tools, projects, chat, notes, CAPEX.
  • Admin — Everything above, plus the Data Management page (upload datasets, trigger ETL).
  • Super Admin — Everything above, plus user creation, role assignment, and system-level settings.

Profile Settings

Click your initials avatar in the top-right corner to open the profile menu. From there you can change your display name, update your password, and adjust notification preferences.

RAN Forecast Dashboard

The dashboard is the home screen. It gives a network-wide view of congestion, sector performance, and CAPEX requirements derived from the latest ingested dataset.

Filter Bar

All dashboard data responds to the four global filters at the top of the page:

FilterOptionsEffect
RegionAll, CENTRAL, EASTERN, NORTHERN, SABAH, SARAWAK, SOUTHERNScopes all metrics and tables to the selected region
YearAvailable years in datasetLocks the data snapshot to that year
WeekISO weeks present in datasetLocks to a specific reporting week
OperatorCelcom, Digi, AllFilters sector-level data by operator

Summary Statistics

Three KPI cards appear below the filters showing Total Sectors, Congested Sectors, and Average Data Volume (GB). A bar below the congested count shows the percentage of congested sectors at a glance.

CAPEX Overview

The CAPEX panel breaks down upgrade costs two ways:

  • By upgrade case — each of the 11 upgrade-case types (e.g. Case 2 Add Bi-Sector, Case 3 Add Network Layer) with sector count and estimated cost.
  • By region — total CAPEX and sector count per geographic region.
  • Top sites table — the sites requiring the most upgrades, sorted by total CAPEX.

Sector Performance Tabs

Three tabs give progressively deeper sector detail:

Sector Performance Metrics — paginated table of every sector with PRB%, throughput, volume, and congestion status. Use the search box to jump to a specific sector ID.

Future Forecasts — ML-predicted PRB%, data volume, or DL throughput for each sector over your chosen horizon (4, 8, 13, or 26 weeks). Expand a sector row to see the full time-series chart.

Congested Sectors — only sectors flagged as congested, with the number of consecutive congested weeks highlighted. Use this for prioritising site-planning actions.

Click Export Combined Dataset at the top of the page to download the full filtered dataset as CSV for offline analysis.

Interactive Map

The Map page is the spatial heart of 3W+. It renders a live MapLibre GL map of Peninsular Malaysia centred on the full network. A toolbar on the left provides all spatial tools.

Coverage Layers

Toggle overlay layers from the Layers panel (top-right of the map):

HeatmapDensity heatmap of congested sectors

Coverage holesClusters of poor-signal drive-test points

Genset / Power Source Routing

The Genset Routing tool finds the nearest substations and electric poles to a selected cell site via road network — not straight-line distance — so the result reflects an accessible cable path.

1
Click a site marker on the map to select it, or use the site-search field to fly to a site by ID.
2
Open the Genset panel from the toolbar. The panel shows Substations and Electric Poles tabs.

The road network is downloaded automatically within a 2.5 km buffer around the site.

3
Results appear sorted by road distance (nearest first). Each result shows the power-source name, road distance in metres, and a polyline drawn on the map tracing the actual road path.
4
Click a result row to highlight that specific path on the map and fly to it.
Only power sources reachable within 2 km by road are returned. Road direction is ignored — the routing treats all roads as bidirectional. Substations are fetched from your uploaded layer (if available) or from OpenStreetMap Overpass.

CCTV Camera Planning

The CCTV tool calculates the optimal placement and field-of-view for surveillance cameras around a site. Click a site, open the CCTV panel, set the camera range and angle parameters, then click Run Analysis. Camera footprints appear as arcs on the map.

Draw & Annotate

The draw toolbar supports four geometry types:

  • Point — drop a marker at any location
  • Line — trace a route or cable path
  • Polygon — define a coverage zone or planning boundary
  • Buffer — place a point and set a radius to generate a circular coverage area

Each annotation can be tagged with a label and attached to a Project so it appears in that project's map view in the Projects module.

Indoor Coverage Simulation

The Sionna RT indoor simulator models signal propagation through a building floor plan. Upload a floor-plan image, place transmitter and receiver points, and run the ray-tracing simulation. The resulting heatmap is overlaid on the floor plan showing predicted signal strength per location.

GeoServer Layers

Admin-uploaded GeoServer layers (WMS/WFS) appear in the Layers panel. These can include district boundaries, planning zones, or operator-specific spatial datasets.

Projects

Projects provides a lightweight kanban board for managing network-planning work items. Each project has tasks and optionally a set of map annotations.

Creating a Project

1
Click + New in the Projects sidebar. Enter a project name and an optional description.
2
The project appears in the sidebar list. Click it to open its detail view (map + kanban board).

Kanban Board

Tasks move across three columns:

To Do

Not yet started. Newly created tasks land here.

In Progress

Actively being worked on.

Done

Completed. Counts toward the project's progress percentage.

Use the Start / Complete / Reopen buttons on each task card to advance its status. Tasks with a due date appear in the Gantt chart below the board.

Rollout Phase Timeline

If a project's tasks follow the naming pattern [P0] Site Selection, [P1] Feasibility Study, … [P10] Go Live, a rollout progress bar appears above the kanban board tracking all 11 standard phases automatically.

Map Annotations

Annotations drawn on the Map page and tagged to a project appear on the project's embedded mini-map. The map zooms to fit all annotations automatically.

06

Chat / AI Agent

The Chat page provides a conversational interface to your network data, powered by LangGraph and Claude. The agent can query DuckDB analytics, search RAG-indexed documents, and generate recommendations.

Example Queries

"Which sectors in the SOUTHERN region have been congested for more than 8 consecutive weeks?"

"What is the total CAPEX required for Case 2 upgrades in Johor?"

"Summarise the key findings from the Q3 network audit report."

RAG Document Search

Documents uploaded to the system (PDF reports, planning notes, audit records) are indexed with pgvector embeddings. The agent automatically retrieves relevant passages when a question touches topics covered in those documents.

The AI agent's answers are generated from your data and should be treated as analytical support, not authoritative decisions. Always verify critical figures against the source dashboards.

Model

In production the agent uses Claude (Anthropic). In local development without an API key it falls back to a local Ollama model automatically. The active model is shown at the top of the chat panel.

Notes

Notes is a lightweight rich-text workspace for storing observations, meeting minutes, or planning decisions alongside your network data.

  • Click + New Note to create a note with a title and freeform body.
  • Notes are stored per-user and are visible only to you.
  • Use notes to document the rationale behind site-planning decisions for future reference.
09

Data Management Admin

The Data page is only visible to Admin and Super Admin users. It controls the ETL pipeline that converts raw network exports into the Parquet files powering all analytics.

Uploading Datasets

1
Select a dataset slot from the upload panel (e.g. Site Coordinates, Congestion Analysis, CAPEX Upgrades).
2
Drag and drop a CSV or Parquet file, or click to browse. Files are validated before processing.
3
Click Run Pipeline (or run individual stages) to ingest the new data. Pipeline status shows per-stage progress and any errors.

Spatial Reference Layers

Upload GeoPackage files for substations, electric poles, and planning boundaries to the appropriate slots. These are used by the Map's Genset Routing and CCTV tools instead of live Overpass API queries.

Running the full pipeline on a large dataset may take several minutes. Do not close the browser tab during pipeline execution.

11

QGIS Road Routing Model

The repository ships a QGIS Processing Model (qgis_models/genset_road_routing.model3) that replicates the platform's Genset Routing logic in a fully offline, QGIS-native environment — no Python backend required.

What It Does

The model finds all road paths from a cell site to nearby substations and electric poles using an undirected road network, then filters to paths under the specified maximum distance (default 2 km) and ranks results nearest-first.

Loading the Model

1
Open QGIS 3.16+. Go to Processing → Toolbox → Models and click the folder icon (Open Existing Model).
2
Browse to qgis_models/genset_road_routing.model3 in the repo. The model appears under MDCI in the toolbox.
3
Double-click the model to open the input dialog. Provide the four inputs listed below.
4
Click Run. The output layer — one path per reachable power source — is added to the QGIS map.

Inputs

ParameterTypeDescription
Cell SitePoint layerSingle-point layer representing the origin cell site
Power SourcesPoint layerSubstations and/or electric poles to route to
Road NetworkLine layerOSM road data (e.g. via QuickOSM) or a local road dataset
Max Distance (m)NumberCut-off road distance in metres. Default: 2000

Output

A line layer Road paths to power sources with these attributes:

  • cost — road distance in metres (map-unit distance)
  • path_rank — integer rank (1 = nearest power source)
  • All attributes from the Power Sources input layer are carried through
# CRS note: use a metric projection for accurate metre distances # EPSG:32647 (UTM Zone 47N) covers Peninsular Malaysia Processing → Toolbox → MDCI → Cell Site to Power Source Road Routing
For a quick road network download, install the QuickOSM QGIS plugin and run a query for highway within a 3 km bounding box around your site. Reproject to EPSG:32647 before running the model.

Troubleshooting

Dashboard shows "No data available"

No dataset has been ingested yet, or the pipeline has not been run. Ask your Admin to upload and process the latest dataset from the Data Management page.

Genset routing returns no results

The road network download from OSM may have timed out, or there are genuinely no substations or poles within 2 km by road. Try a site in a more urban area. If failures are consistent, ask Admin to upload a local substation GeoPackage to reduce Overpass dependency.

Chat agent gives outdated answers

The agent's knowledge is limited to the currently ingested dataset and indexed documents. Ensure the latest data has been processed and that relevant documents have been uploaded for RAG indexing.

Map layers not loading

Coverage tile layers require an internet connection to the tile CDN. GeoServer layers require the GeoServer container to be running. Check with your Admin if the backend services are up (docker compose ps).

Login fails with "Invalid credentials"

Check that Caps Lock is off. If you have forgotten your password, ask a Super Admin to reset it from the Super Admin page.
